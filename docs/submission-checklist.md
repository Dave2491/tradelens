# Submission Checklist

Use this before final Bitget AI Base Camp submission.

## Repo

- README explains the product clearly.
- `docs/` folder explains evidence, data sources, demo flow, and sample signals.
- `.env` is not committed.
- `.env.example` contains only placeholder environment variables.
- `ROADMAP.md` is not committed.
- Tests pass.
- Production build passes.

## Demo

- Record Signal Desk.
- Paste a fresh signal.
- Show live Bitget market data loading.
- Show Qwen AI review.
- Show original signal vs TradeLens plan.
- Paper-track original and safer plans.
- Open Trade Monitor.
- Open Evidence page.
- Export evidence JSON.

## Screenshots

Capture:

- Signal Desk
- live trade map
- AI review
- plan comparison
- Trade Monitor
- Evidence page
- exported JSON file name or preview

## Deployment

- Vercel project is linked.
- Qwen environment variables are added to Vercel.
- Production deployment succeeds.
- Live URL can analyze a signal.
- Live URL can export evidence.

## Final submission assets

- GitHub repo link
- deployed app link
- demo video link
- screenshot set
- evidence JSON export
- progress/social post link, if required
